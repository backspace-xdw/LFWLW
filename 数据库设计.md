# 物联网远程监控管理Web平台 - 数据库设计

## 1. 数据库架构概述

### 1.1 数据库选型
- **PostgreSQL 15**: 主业务数据库，存储设备信息、用户数据、配置等
- **InfluxDB 2.0**: 时序数据库，存储设备实时数据、历史数据
- **Redis 7.0**: 缓存数据库，存储会话、热点数据、实时状态
- **MinIO/S3**: 对象存储，存储3D模型、文档、图片等文件

### 1.2 数据分区策略
- 业务数据：按租户ID分区，支持多租户隔离
- 时序数据：按时间分区，自动归档历史数据
- 缓存数据：按过期时间自动清理
- 文件数据：按类型和日期分桶存储

## 3. InfluxDB时序数据库设计

### 3.1 Measurement设计

#### device_data（设备数据）
```influxql
-- Measurement: device_data
-- Tags: device_id, point_id, data_type
-- Fields: value, quality, timestamp
-- Retention Policy: 
--   - raw: 7 days (原始数据)
--   - hourly: 30 days (小时聚合)
--   - daily: 1 year (天聚合)
--   - monthly: forever (月聚合)

CREATE DATABASE iot_platform;

USE iot_platform;

-- 创建保留策略
CREATE RETENTION POLICY "raw" ON "iot_platform" DURATION 7d REPLICATION 1 DEFAULT;
CREATE RETENTION POLICY "hourly" ON "iot_platform" DURATION 30d REPLICATION 1;
CREATE RETENTION POLICY "daily" ON "iot_platform" DURATION 365d REPLICATION 1;
CREATE RETENTION POLICY "monthly" ON "iot_platform" DURATION INF REPLICATION 1;

-- 创建连续查询进行数据聚合
-- 小时聚合
CREATE CONTINUOUS QUERY "cq_hourly" ON "iot_platform"
BEGIN
  SELECT mean("value") AS mean_value, 
         max("value") AS max_value, 
         min("value") AS min_value,
         count("value") AS count_value
  INTO "hourly"."device_data"
  FROM "device_data"
  GROUP BY time(1h), device_id, point_id
END;

-- 天聚合
CREATE CONTINUOUS QUERY "cq_daily" ON "iot_platform"
BEGIN
  SELECT mean("mean_value") AS mean_value,
         max("max_value") AS max_value,
         min("min_value") AS min_value,
         sum("count_value") AS count_value
  INTO "daily"."device_data"
  FROM "hourly"."device_data"
  GROUP BY time(1d), device_id, point_id
END;
```

#### device_status（设备状态）
```influxql
-- Measurement: device_status
-- Tags: device_id, status_type
-- Fields: status, message

CREATE CONTINUOUS QUERY "cq_status_daily" ON "iot_platform"
BEGIN
  SELECT last("status") AS last_status,
         count("status") AS status_changes
  INTO "daily"."device_status"
  FROM "device_status"
  GROUP BY time(1d), device_id, status_type
END;
```

## 4. Redis缓存设计

### 4.1 Key设计规范

```javascript
// Key命名规范: prefix:type:id:field

// 用户会话
session:{sessionId} // TTL: 30min

// 用户权限缓存
user:permissions:{userId} // TTL: 5min

// 设备状态
device:status:{deviceId} // TTL: 10s
device:realtime:{deviceId}:{pointId} // TTL: 5s

// 告警计数
alarm:count:{severity} // TTL: 1min
alarm:active:{deviceId} // Set类型，活动告警ID集合

// 分布式锁
lock:device:{deviceId} // TTL: 30s
lock:scene:{sceneId} // TTL: 60s

// 数据订阅
sub:device:{deviceId} // Set类型，订阅者列表
sub:channel:{channelId} // List类型，消息队列

// API限流
rate:api:{userId}:{api} // TTL: 1min
rate:global:{api} // TTL: 1min

// 统计数据
stats:online:devices // String类型，在线设备数
stats:active:users // HyperLogLog类型，活跃用户
stats:api:calls // Hash类型，API调用统计
```

```javascript
// 设备实时数据 (Hash)
```

### 6.3 查询优化

```sql
-- 使用EXPLAIN ANALYZE优化查询
EXPLAIN (ANALYZE, BUFFERS) 
SELECT d.*, COUNT(a.id) as active_alarms
FROM devices d
LEFT JOIN alarms a ON d.id = a.device_id AND a.status = 'active'
WHERE d.org_id = ? AND d.status = 'online'
GROUP BY d.id;

-- 使用物化视图加速统计查询
CREATE MATERIALIZED VIEW device_statistics AS
SELECT 
    org_id,
    COUNT(*) as total_devices,
    COUNT(*) FILTER (WHERE status = 'online') as online_devices,
    COUNT(*) FILTER (WHERE status = 'offline') as offline_devices
FROM devices
WHERE deleted_at IS NULL
GROUP BY org_id;

-- 定期刷新物化视图
REFRESH MATERIALIZED VIEW CONCURRENTLY device_statistics;
```

## 7. 数据备份与恢复

### 7.1 备份策略

## 8. 数据安全

### 8.1 加密存储

```sql
-- 使用pgcrypto扩展加密敏感数据
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- 加密存储密码
INSERT INTO users (username, password_hash) 
VALUES ('user@example.com', crypt('password123', gen_salt('bf')));

-- 验证密码
SELECT * FROM users 
WHERE username = 'user@example.com' 
AND password_hash = crypt('password123', password_hash);

-- 加密存储敏感配置
UPDATE system_config 
SET value = pgp_sym_encrypt(value, 'encryption_key')
WHERE is_encrypted = TRUE;

-- 解密读取
SELECT pgp_sym_decrypt(value::bytea, 'encryption_key') as value
FROM system_config 
WHERE key = 'smtp_password';
```

### 8.2 访问控制

```sql
-- 创建只读用户
CREATE USER readonly_user WITH PASSWORD 'secure_password';
GRANT CONNECT ON DATABASE iot_platform TO readonly_user;
GRANT USAGE ON SCHEMA public TO readonly_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly_user;

-- 创建应用用户
CREATE USER app_user WITH PASSWORD 'app_password';
GRANT CONNECT ON DATABASE iot_platform TO app_user;
GRANT USAGE ON SCHEMA public TO app_user;
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;
GRANT USAGE ON ALL SEQUENCES IN SCHEMA public TO app_user;

-- 行级安全策略
ALTER TABLE devices ENABLE ROW LEVEL SECURITY;

CREATE POLICY devices_org_policy ON devices
    FOR ALL
    TO app_user
    USING (org_id IN (
        SELECT org_id FROM user_organizations 
        WHERE user_id = current_setting('app.current_user_id')::uuid
    ));
```
