# 远程登录问题解决方案

## 问题描述
- 前端页面可以访问（http://67.169.169.65:50000）
- 但登录时API请求被发送到错误的地址：`http://ljinvestment.diskstation.me:50001`
- 导致登录失败

## 根本原因
前端应用在某处缓存或硬编码了API地址，需要清除。

## 解决方法

### 方法1：使用无痕模式（最快）
1. 打开浏览器的无痕/隐私模式
2. 访问 http://67.169.169.65:50000
3. 使用 admin/admin123 登录

### 方法2：清除浏览器数据
1. 打开 http://67.169.169.65:50000
2. 按 F12 打开开发者工具
3. 右键点击刷新按钮，选择"清空缓存并硬性重新加载"
4. 或在 Application 标签中清除：
   - Local Storage
   - Session Storage
   - Cookies

### 方法3：使用测试页面
访问我创建的修复页面：
http://67.169.169.65:50000/fix_remote_login.html

### 验证修复成功
1. 在开发者工具的 Network 标签中
2. 登录时应该看到请求发送到：
   - `/api/v1/auth/login` （相对路径）
   - 而不是完整的外部URL

## 长期解决方案
需要确保前端代码中没有硬编码的API地址，所有请求都应该使用相对路径。
