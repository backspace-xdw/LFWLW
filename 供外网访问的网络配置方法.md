# 供外网访问的网络配置方法

> 本文档提供了一套完整的解决方案，用于将局域网内的Web项目配置为可通过外网访问。

## 📋 前置条件

- 本地服务器IP：192.168.0.20
- 外网IP：67.169.169.65
- 已安装nginx
- 具有路由器管理权限

## 🏗️ 整体架构

```
[外网用户] → [外网IP:端口] → [路由器端口转发] → [nginx反向代理:80] → [本地项目:实际端口]
```

## 📝 配置步骤

### 步骤1：确定项目端口

假设您的新项目运行在：
- 前端：8080端口
- 后端：8081端口（如果有）

### 步骤2：创建nginx配置文件

创建文件：`/etc/nginx/sites-available/项目名称`

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为您的域名或IP
    
    # 前端代理
    location / {
        proxy_pass http://127.0.0.1:8080;  # 替换为实际前端端口
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host localhost:8080;  # 重要：绕过安全限制
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
        proxy_cache_bypass $http_upgrade;
        
        # 开发环境配置
        proxy_read_timeout 86400;
        proxy_send_timeout 86400;
    }
    
    # 后端API代理（如果需要）
    location /api {
        proxy_pass http://127.0.0.1:8081;  # 替换为实际后端端口
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # WebSocket支持（如果需要）
    location /ws {
        proxy_pass http://127.0.0.1:8081;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    # 文件上传大小限制
    client_max_body_size 50M;
    
    # 超时设置
    proxy_connect_timeout 600s;
    proxy_send_timeout 600s;
    proxy_read_timeout 600s;
}
```

### 步骤3：使用不同端口避免冲突

如果80端口已被占用，可以使用其他端口：

```nginx
server {
    listen 8090;  # 使用不同端口
    server_name _;
    
    # ... 其他配置相同
}
```

### 步骤4：启用nginx配置

```bash
# 创建符号链接
sudo ln -sf /etc/nginx/sites-available/项目名称 /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载nginx
sudo systemctl reload nginx
```

### 步骤5：配置路由器端口转发

在路由器管理界面添加规则：

| 外部端口 | 内部IP | 内部端口 | 协议 | 说明 |
|---------|--------|---------|------|------|
| 选择未占用端口（如8090） | 192.168.0.20 | 80或8090 | TCP | 项目名称 |

### 步骤6：防火墙配置（如需要）

```bash
# 允许端口
sudo ufw allow 80/tcp
sudo ufw allow 8090/tcp  # 如果使用其他端口

# 查看状态
sudo ufw status
```

## 🔧 自动化脚本

### 创建配置脚本：`setup_external_access.sh`

```bash
#!/bin/bash

# 配置变量
PROJECT_NAME="$1"
FRONTEND_PORT="$2"
BACKEND_PORT="$3"
NGINX_PORT="${4:-80}"  # 默认80端口

if [ $# -lt 3 ]; then
    echo "使用方法: ./setup_external_access.sh 项目名称 前端端口 后端端口 [nginx端口]"
    echo "示例: ./setup_external_access.sh myproject 8080 8081 8090"
    exit 1
fi

echo "=== 配置外网访问: $PROJECT_NAME ==="
echo "前端端口: $FRONTEND_PORT"
echo "后端端口: $BACKEND_PORT"
echo "Nginx端口: $NGINX_PORT"
echo ""

# 创建nginx配置
cat > /tmp/${PROJECT_NAME}-nginx.conf << EOF
server {
    listen $NGINX_PORT;
    server_name _;
    
    location / {
        proxy_pass http://127.0.0.1:$FRONTEND_PORT;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host localhost:$FRONTEND_PORT;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
        proxy_set_header X-Forwarded-Host \$host;
        proxy_cache_bypass \$http_upgrade;
        proxy_read_timeout 86400;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:$BACKEND_PORT;
        proxy_http_version 1.1;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
    }
    
    location /ws {
        proxy_pass http://127.0.0.1:$BACKEND_PORT;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host \$host;
    }
    
    client_max_body_size 50M;
    proxy_connect_timeout 600s;
    proxy_send_timeout 600s;
    proxy_read_timeout 600s;
}
EOF

echo "Nginx配置已生成到: /tmp/${PROJECT_NAME}-nginx.conf"
echo ""
echo "请执行以下命令完成配置："
echo "1. sudo cp /tmp/${PROJECT_NAME}-nginx.conf /etc/nginx/sites-available/$PROJECT_NAME"
echo "2. sudo ln -sf /etc/nginx/sites-available/$PROJECT_NAME /etc/nginx/sites-enabled/"
echo "3. sudo nginx -t"
echo "4. sudo systemctl reload nginx"
echo ""
echo "然后在路由器添加端口转发："
echo "外部端口: 选择未占用的端口"
echo "内部IP: 192.168.0.20"
echo "内部端口: $NGINX_PORT"
echo "协议: TCP"
```

### 创建快速检查脚本：`check_external_access.sh`

```bash
#!/bin/bash

echo "=== 外网访问配置检查 ==="
echo ""

echo "1. 本地网络信息："
ip addr show | grep "inet " | grep -v "127.0.0.1"
echo ""

echo "2. Nginx状态："
sudo systemctl status nginx --no-pager | head -5
echo ""

echo "3. 端口监听状态："
sudo netstat -tlnp | grep -E "nginx|node|python|java" | grep -v "127.0.0.1"
echo ""

echo "4. 已配置的nginx站点："
ls -la /etc/nginx/sites-enabled/
echo ""

echo "5. 访问地址："
echo "外网IP: 67.169.169.65"
echo "请确保路由器已配置端口转发"
```

## 🚀 快速配置示例

假设新项目是一个Vue+Express项目：
- Vue前端运行在：3000端口
- Express后端运行在：3001端口
- 想要通过外网8888端口访问

执行步骤：
```bash
# 1. 运行配置脚本
./setup_external_access.sh vue-project 3000 3001 8888

# 2. 应用nginx配置
sudo cp /tmp/vue-project-nginx.conf /etc/nginx/sites-available/vue-project
sudo ln -sf /etc/nginx/sites-available/vue-project /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 3. 在路由器配置
外部端口: 8888
内部IP: 192.168.0.20
内部端口: 8888
协议: TCP

# 4. 访问地址
http://67.169.169.65:8888
```

## ⚠️ 重要注意事项

### 1. 前端项目配置
- **React/Vue项目**：确保没有在环境变量中硬编码API地址
- **开发服务器**：某些框架（如Vite）需要特殊的Host头配置

### 2. 后端项目配置
- **CORS设置**：开发环境设置为 `*` 或具体的外网地址
- **API路径**：使用相对路径，不要硬编码域名

### 3. 安全建议
- 仅在开发环境使用此配置
- 生产环境需要额外的安全措施
- 考虑使用HTTPS和认证机制

### 4. 故障排查
```bash
# 检查服务是否运行
ps aux | grep [项目关键词]

# 检查端口是否监听
netstat -tlnp | grep [端口号]

# 查看nginx错误日志
sudo tail -f /var/log/nginx/error.log

# 测试本地访问
curl http://localhost:[端口]
```
