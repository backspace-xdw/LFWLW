# 告警管理系统使用指南

## 概述
告警管理系统提供了全面的工业告警功能，支持多种告警类型、灵活的规则配置和完整的告警生命周期管理。

## 告警类型

### 1. 限值告警
- **高高报警（HH）**: 
  - 严重级别（Critical）
  - 测量值超过高高限值时触发
  - 通常用于设备保护和紧急停机
  
- **高报警（H）**: 
  - 高级别（High）
  - 测量值超过高限值时触发
  - 用于提前预警

- **低报警（L）**: 
  - 中级别（Medium）
  - 测量值低于低限值时触发
  - 用于资源不足预警

- **低低报警（LL）**: 
  - 严重级别（Critical）
  - 测量值低于低低限值时触发
  - 通常触发保护动作

### 2. 动态告警
- **变化率报警（ROC）**: 
  - 监测参数在单位时间内的变化量
  - 用于检测异常快速变化
  - 可配置计算周期（1-60分钟）

- **偏差报警（DEVIATION）**: 
  - 监测实际值与设定值的偏差
  - 用于过程控制质量监控

## 告警规则配置

### 基本参数
- **规则名称**: 描述性名称
- **设备参数**: 选择要监控的设备和参数
- **告警类型**: 选择六种类型之一
- **阈值**: 触发告警的数值
- **严重级别**: 自动根据类型设置

### 高级参数
- **延迟时间**: 
  - 参数超限后延迟触发的时间（秒）
  - 避免瞬时波动造成误报
  - 范围：0-300秒

- **死区**: 
  - 告警恢复的缓冲区间
  - 避免在阈值附近频繁告警
  - 对于高限告警：值 < 阈值 - 死区 时恢复
  - 对于低限告警：值 > 阈值 + 死区 时恢复

- **计算周期**: 
  - 仅用于ROC类型
  - 变化率计算的时间窗口
  - 范围：1-60分钟

## 告警管理功能

### 1. 告警列表
- **实时更新**: WebSocket推送新告警
- **状态筛选**: 活动/已确认/已解决
- **多维度过滤**: 
  - 按严重程度
  - 按设备
  - 按时间范围
  - 关键词搜索

### 2. 告警处理
- **确认告警**: 
  - 表示已知晓并开始处理
  - 需要填写处理说明
  - 记录确认人和时间

- **解决告警**: 
  - 表示问题已解决
  - 自动记录解决时间
  - 可添加解决说明

### 3. 告警统计
- 总告警数量
- 各状态告警数量
- 各级别告警分布
- 实时更新的统计卡片

## 使用流程

### 配置告警规则
1. 点击"规则配置"按钮
2. 点击"新增规则"
3. 填写规则信息：
   - 选择设备和参数
   - 选择告警类型
   - 设置阈值
   - 配置延迟和死区
4. 保存规则

### 处理告警
1. 在告警列表查看活动告警
2. 点击"确认"按钮
3. 填写处理措施
4. 确认提交
5. 问题解决后点击"解决"

### 查看告警详情
1. 点击告警记录的"详情"按钮
2. 查看完整信息：
   - 告警参数和阈值
   - 触发时间和数值
   - 处理历史记录
   - 相关人员信息

## API接口

### 告警规则管理
```
GET    /api/v1/alarm-rules      # 获取所有规则
POST   /api/v1/alarm-rules      # 创建规则
PUT    /api/v1/alarm-rules/:id  # 更新规则
DELETE /api/v1/alarm-rules/:id  # 删除规则
```

### WebSocket事件
```
监听事件:
- alarm:new     # 新告警
- alarm:update  # 告警更新

订阅频道:
- alarms:all         # 所有告警
- alarms:critical    # 严重告警
- alarms:high        # 高级告警
```

## 最佳实践

### 1. 规则设置建议
- 使用分级告警（L→H→HH）提供预警
- 设置合理的延迟时间（5-30秒）
- 死区一般设为阈值的2-5%
- ROC周期根据过程特性选择

### 2. 告警响应
- 及时确认告警避免遗漏
- 详细记录处理过程
- 定期分析告警趋势
- 优化规则减少误报

### 3. 系统维护
- 定期审查告警规则
- 清理过期的历史告警
- 监控告警系统性能
- 备份重要规则配置

## 故障排除

### 告警不触发
1. 检查规则是否启用
2. 确认设备数据正常
3. 查看延迟设置
4. 验证阈值配置

### 频繁告警
1. 增加延迟时间
2. 扩大死区范围
3. 调整阈值
4. 使用ROC替代限值告警

### 性能问题
1. 减少实时数据频率
2. 优化规则数量
3. 清理历史数据
4. 检查网络连接
