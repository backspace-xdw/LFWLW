## 测试场景

### 1. 登录测试

#### 1.1 正常登录测试
**步骤**:
1. 访问登录页面
2. 输入用户名: admin
3. 输入密码: Lfwlw@2024
4. 点击登录按钮

**预期结果**:
- 登录成功，跳转到仪表板页面
- 右上角显示用户信息
- 获得有效的JWT token

#### 1.2 错误密码测试
**步骤**:
1. 访问登录页面
2. 输入用户名: admin
3. 输入错误密码: wrongpassword
4. 点击登录按钮

**预期结果**:
- 显示错误提示："登录失败，请检查用户名和密码"
- 停留在登录页面
- 记录失败尝试次数

#### 1.3 账号锁定测试
**步骤**:
1. 连续5次使用错误密码尝试登录
2. 第6次尝试登录

**预期结果**:
- 账号被锁定
- 显示提示："账号已被锁定，请联系管理员"
- 即使使用正确密码也无法登录

4. 点击"创建"按钮

**预期结果**:
- 弹出成功提示框，显示初始密码
- 初始密码格式: Lfwlw@2024XXXX (4位随机字符)
- 用户列表中出现新用户
- 新用户状态为"未激活"

#### 2.2 角色权限限制测试
**前置条件**: 使用operator账号登录

**步骤**:

**预期结果**:

### 3. 首次登录密码修改测试

#### 3.1 新用户首次登录
**步骤**:
1. 使用新创建的用户登录
   - 用户名: operator001
   - 密码: (使用创建时显示的初始密码)
2. 登录成功后自动弹出修改密码弹窗

**预期结果**:
- 成功登录但不跳转页面
- 自动显示"首次登录 - 修改密码"弹窗
- 弹窗无法关闭，必须修改密码

#### 3.2 密码策略验证
**步骤**:
1. 在修改密码弹窗中尝试输入不符合要求的密码:
   - 尝试: "123456" (无大写字母)
   - 尝试: "Abcdefg" (无数字)
   - 尝试: "Abc123" (长度不足8位)

**预期结果**:
- 每次都显示验证错误提示
- 提示内容: "密码至少8位，必须包含大写字母、小写字母和数字"

#### 3.3 成功修改初始密码
**步骤**:
1. 输入符合要求的新密码: "Welcome@123"
2. 确认新密码: "Welcome@123"
3. 点击确定

**预期结果**:
- 显示"密码修改成功，请使用新密码登录"
- 自动退出到登录页面
- 使用新密码可以成功登录
- 用户状态变为"正常"

### 4. 用户管理操作测试

#### 4.1 编辑用户信息
**前置条件**: 使用admin账号登录

**步骤**:
1. 在用户列表找到operator001
2. 点击编辑按钮
3. 修改信息:
   - 姓名: 张三 → 李四
   - 部门: 生产部 → 技术部
   - 有效期至: 设置为30天后
4. 点击保存

**预期结果**:
- 用户信息更新成功
- 列表中显示更新后的信息
- 有效期显示具体日期

#### 4.2 重置用户密码
**步骤**:
1. 点击用户的重置密码按钮
2. 确认重置操作

**预期结果**:
- 弹出成功提示，显示新的临时密码
- 用户下次登录时需要修改密码
- 用户状态保持不变

#### 4.3 锁定/解锁账号
**步骤**:
1. 点击锁定按钮，确认锁定
2. 尝试使用该账号登录
3. 返回管理页面，点击解锁按钮

**预期结果**:
- 锁定后状态显示"已锁定"
- 被锁定账号无法登录
- 解锁后可以正常登录

### 5. 会话管理测试

#### 5.1 会话超时测试
**步骤**:
1. 登录系统
2. 保持页面打开但不操作30分钟
3. 30分钟后尝试操作

**预期结果**:
- 自动跳转到登录页面
- 显示会话超时提示
- 需要重新登录

#### 5.2 Token刷新测试
**步骤**:
1. 登录系统
2. 持续操作超过30分钟

**预期结果**:
- 会话保持活跃，不会超时
- Token在后台自动刷新
- 操作不受影响
