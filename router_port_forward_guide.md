# 路由器端口转发配置指南

## 配置目标
将外网访问 `http://67.169.169.65:8080` 转发到内网 `192.168.0.20:80`

## 步骤一：登录路由器管理界面

1. 打开浏览器，访问路由器管理地址（常见地址）：
   - `http://192.168.0.1`
   - `http://192.168.1.1`
   - `http://192.168.0.254`
   - `http://10.0.0.1`

2. 输入管理员账号密码（查看路由器背面标签）

## 步骤二：找到端口转发设置

根据路由器品牌，端口转发功能可能在以下位置：

### TP-Link路由器
- 应用管理 → 虚拟服务器
- 转发规则 → 虚拟服务器
- 高级设置 → NAT转发 → 虚拟服务器

### 华为路由器
- 更多功能 → 安全设置 → NAT服务 → 端口映射
- 高级设置 → NAT → 端口映射

### 小米路由器
- 常用设置 → 端口转发
- 高级设置 → 端口转发

### ASUS路由器
- WAN → Virtual Server / Port Forwarding
- 外部网络 → 端口转发

### Netgear路由器
- Dynamic DNS → Port Forwarding
- 高级 → 高级设置 → 端口转发/端口触发

## 步骤三：添加端口转发规则

创建新规则，填写以下信息：

```
规则名称：LFWLW系统
外部端口：8080
内部IP地址：192.168.0.20
内部端口：80
协议：TCP（或TCP/UDP）
状态：启用
```

### 示例配置表：
| 设置项 | 值 |
|--------|-----|
| 服务名称 | LFWLW |
| 外部端口起始 | 8080 |
| 外部端口结束 | 8080 |
| 内部IP地址 | 192.168.0.20 |
| 内部端口起始 | 80 |
| 内部端口结束 | 80 |
| 协议 | TCP |
| 启用 | ✓ |

## 步骤四：保存并重启路由器

1. 点击"保存"或"应用"按钮
2. 部分路由器需要重启才能生效
3. 等待路由器重启完成（约1-2分钟）

## 步骤五：验证配置

### 本地验证
```bash
# 检查本地nginx是否正常
curl http://192.168.0.20
```

### 外网验证
让同事访问：`http://67.169.169.65:8080`

## 常见问题

### 1. 无法访问
- 检查Windows/Linux防火墙是否允许80端口
- 确认nginx服务正在运行
- 确认内网IP地址正确（192.168.0.20）

### 2. 端口被占用
如果8080端口已被占用，可以使用其他端口：
- 8081, 8090, 9000, 9090等

### 3. 安全提醒
- 端口转发会暴露内网服务到公网
- 建议仅在开发测试时使用
- 生产环境需要额外的安全措施

## 防火墙配置（如需要）

### Ubuntu/Debian
```bash
sudo ufw allow 80/tcp
sudo ufw reload
```

### CentOS/RHEL
```bash
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
```
